# v1.0 client integration guide

> Single doc covering everything a client needs to integrate against
> YgLicenseServer v1.0+. Supersedes the ASM-specific
> [`v1.0-asm-client-handoff.md`](v1.0-asm-client-handoff.md) (which only
> covered the `aud` claim). Read this for new integrations; the ASM
> handoff still applies as a worked example.

## TL;DR — what changed in v1.0

| Concern | Pre-v1.0 | v1.0+ |
|---|---|---|
| JWT decoding | `jwt.decode(token, pub, algorithms=...)` worked without `audience=` | `aud = product.slug` is now in every payload; clients MUST pass `audience=product_slug` or ignore aud explicitly |
| JWT header | No `kid` | `kid = product.id` (opaque); lets multi-product clients pick the right key without trying each one |
| License key on `/v1/check` | Plaintext key persisted on every row + returned in admin responses | Server stores a peppered hash + a truncated display form; plaintext is shown exactly ONCE at issuance |
| `webhook_secret` on `/v1/check` | Echoed for any license that had a URL | Only echoed when `webhook_url_source == 'self'` (i.e. the client self-registered the URL) |
| Webhook URL set by admin and client sends differing `public_url` | Heartbeat succeeded; URL silently overwritten (security bug, fixed in v1.0.0) | v1.0.0–v1.0.2: heartbeat 409s. **v1.0.3+: heartbeat succeeds, URL unchanged, audit event logged.** |

## 1. JWT verification — `aud`

Every JWT minted by `/v1/check` from v1.0.0 onwards carries `aud =
product.slug`. PyJWT (and many other libraries) reject tokens with an
`aud` claim unless the caller passes a matching `audience=` parameter.

**Python (pyjwt):**

```python
import jwt

claims = jwt.decode(
    token, public_key_pem,
    algorithms=["EdDSA"],
    audience=PRODUCT_SLUG,             # e.g. "asm"
    options={"verify_exp": False},     # let valid_until be the source of truth
)
```

**Rolling-upgrade fallback** (for clients with cached pre-v1.0 tokens):

```python
try:
    claims = jwt.decode(token, pub, algorithms=["EdDSA"],
                        audience=PRODUCT_SLUG, options={"verify_exp": False})
except jwt.MissingRequiredClaimError:
    # Pre-v1.0 token: no aud. Verify everything else, skip aud only.
    claims = jwt.decode(token, pub, algorithms=["EdDSA"],
                        options={"verify_exp": False, "verify_aud": False})
```

Drop the fallback once every install has phoned home post-v1.0.

**Dart (`dart_jsonwebtoken` / similar):** if your library's missing-aud
behavior is unstable across versions, validate `aud` in user-space:
decode without aud verification, then check `payload['aud'] != null
&& payload['aud'] != PRODUCT_SLUG` → reject.

## 2. JWT header — `kid`

The header now includes `kid = product.id` (opaque UUID, NOT the
product slug). Single-product clients can ignore it. Multi-product
clients (rare — most installs target one product) can use it to pick
the right public key from a `Map<kid, pubkey>` without try/except.

The product's pub key is at `GET /v1/products/{slug}/pubkey` — no auth.

## 3. License key storage on the server

Plaintext keys are no longer stored. The server has:

- `key_hash` (peppered BLAKE2b-256) — used by `/v1/check` for lookups
- `key_display` (`<prefix>_<first6>…<last4>`) — used in admin UI + CSV
  exports

Operational implications:

- Issuance HTTP response, the customer email, and the admin-UI flash
  banner are the **only** times the plaintext appears. Lose it then
  and the only recovery is re-issuing the license.
- The deprecated `licenses.key` column is retained in v1.0.x for
  in-place rollback safety. It WILL be dropped in v1.1; do not write
  client code that reads it via admin API.

## 4. Webhook URL provenance — admin-set vs self-registered

The `webhook_url_source` column (`'admin'` | `'self'`) governs two
policies:

### 4a. Secret echo on `/v1/check`

`source='admin'` — `/v1/check` response field `webhook_secret` is
always `null`. The customer-side receiver must already have the secret
(distributed out-of-band via the admin-UI one-time display).

`source='self'` — `webhook_secret` is echoed on every `/v1/check`
response. The client can persist it (e.g. Firestore, local file) and
the customer-side receiver uses it for HMAC verification.

### 4b. `public_url` override

`source='admin'` — `/v1/check` requests with a `public_url` that
differs from the stored URL are ignored. v1.0.0–v1.0.2 returned 409
and broke the heartbeat; **v1.0.3+ keeps the heartbeat alive** and
logs an `webhook:override_refused` event. URL is never overwritten.

`source='self'` — `/v1/check` requests with `public_url` upsert the
URL. First time triggers secret minting; subsequent identical requests
are no-ops.

### 4c. When to use which

| Scenario | Use |
|---|---|
| Customer-managed receiver with a stable URL (their own infra) | `admin` — admin sets URL via UI, distributes secret out-of-band |
| Client-managed receiver with a dynamic URL (cloud function, tunnel, per-install endpoint) | `self` — client sends `public_url` on every `/v1/check`, persists the echoed `webhook_secret` |
| Customer-managed but the URL might change | `self` — let the client self-register on every boot, accept whatever URL the client reports |

Most real integrations want `self`. The `admin` mode exists for the
case where the operator wants to control where webhooks land and trusts
the customer's receiver to be stable.

### 4d. Migrating an admin-set license to self

v1.0.3 ships a one-click button in the admin UI's license-edit modal:
**Convert to self**. It:

1. Keeps the URL unchanged.
2. Flips `webhook_url_source` to `'self'`.
3. Rotates the signing secret (the new one will echo on next
   `/v1/check`; the old one is dead).
4. Logs `webhook:converted_to_self`.

After the button: the client's next `/v1/check` echoes the new
secret. The client persists it and starts using it for HMAC
verification.

Pre-v1.0.3 the workaround was: admin clears the URL → client
self-registers same URL on next heartbeat → done. Brief window
where the URL is unset on LS; mostly OK for steady-state but
race-prone for high-frequency push channels.

## 5. `/v1/check` request shape (recap)

```json
POST /v1/check
{
  "key": "asm_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
  "install_id": "stable-uuid-per-machine",
  "version": "1.2.3",
  "public_url": "https://customer.example.com/webhooks/license"
}
```

Response (200):

```json
{
  "jwt": "...",
  "valid_until": "2027-05-22T00:00:00Z",
  "features": {"premium": true},
  "max_users": 10,
  "license_id": "uuid",
  "product": "asm",
  "webhook_secret": "...or null"
}
```

`public_url` is **optional**. Send it if the client owns its receiver
URL (per-install tunnels, per-tenant cloud functions). Skip it
otherwise.

## 6. Heartbeat cadence

Recommended: every 6 hours, plus on every app boot. The server's grace
window (`LICENSE_GRACE_DAYS`, default 7) lets you ride out network
outages without locking customers out.

Cache the JWT and use it offline until `valid_until` + grace. Re-check
on demand if a feature gate is failing and the cached token might be
stale.

## 7. Common integration mistakes

- **Forgetting to pass `audience=`** after upgrading the LS to v1.0.
  `InvalidAudienceError` on every `/v1/check` consumer-side decode.
- **Persisting the echoed `webhook_secret` when the license is
  admin-set.** The field will be `null` from v1.0.0 onwards — the
  client must have been given the secret out-of-band.
- **Hardcoding a `public_url` on a client that talks to multiple LS
  installs.** If one of those installs has an admin-set URL pointing
  somewhere else, v1.0.0–v1.0.2 broke the whole heartbeat. v1.0.3+
  is tolerant.
- **Treating `key_display` (truncated) as a real key.** It's a UI
  label; `/v1/check` won't accept it.

## Reference implementations

- ASM (Python, FastAPI) — `backend/app/licensing/` in
  [AnimalShelterManager](https://github.com/Why-Gee/AnimalShelterManager).
  Handles `aud`, kid, cached JWT with grace, license guard middleware.
- WorkoutTracker (Dart, Flutter) — `lib/licensing/` in
  [YgWorkoutTracker](https://github.com/Why-Gee/YgWorkoutTracker).
  Handles user-space `aud` check (Dart library compat), self-register
  via `public_url`, Firestore-persisted webhook secret.
