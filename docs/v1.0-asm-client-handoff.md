# ASM client handoff — v1.0 JWT `aud` claim

> Created by the YgLicenseServer v1.0.0 release pass on 2026-05-22.
> Read alongside `CHANGELOG.md` in this repo for the full breaking-change list.

## What changed on the server side

YgLicenseServer v1.0.0 is now live at `https://yg-license-server.duckdns.org`. Every JWT minted by `/v1/check` now carries an `aud = product.slug` claim. `pyjwt` validates `aud` whenever the token contains it — so any client decoding via:

```python
jwt.decode(token, public_key_pem, algorithms=["EdDSA"], options={"verify_exp": False})
```

— without passing `audience=` — will raise `jwt.InvalidAudienceError` and fail every token verification.

## What ASM needs to change

Two callsites in `backend/app/licensing/` need `audience="asm"` added:

### 1. `backend/app/licensing/middleware.py:54`

Current:

```python
return jwt.decode(
    token,
    _public_key_pem(),
    algorithms=["EdDSA"],
    options={"verify_exp": False},
)
```

After:

```python
return jwt.decode(
    token,
    _public_key_pem(),
    algorithms=["EdDSA"],
    audience="asm",
    options={"verify_exp": False},
)
```

### 2. `backend/app/licensing/feature_flags.py:56`

Current:

```python
return jwt.decode(
    token, _public_key_pem(), algorithms=["EdDSA"], options={"verify_exp": False}
)
```

After:

```python
return jwt.decode(
    token, _public_key_pem(), algorithms=["EdDSA"], audience="asm",
    options={"verify_exp": False},
)
```

## Verification

After landing the change:

1. Run ASM's full test suite. Any test that mocks a JWT for license verification must also include `"aud": "asm"` in the payload, otherwise it will start failing.
2. Local smoke: start ASM with a valid license key against the prod server and confirm `/health` (or whatever ASM exposes) still reports the license as valid.
3. Production smoke: tail ASM logs after upgrade; an `InvalidAudienceError` in the licensing middleware means the change didn't land where it should.

## Why not just use a constant?

`"asm"` is the product slug used by every ASM install. If you want to centralise:

```python
# backend/app/licensing/__init__.py (or constants.py)
PRODUCT_SLUG = "asm"
```

Then import in both callsites. Not required — the inline string literal is fine for a two-callsite change.

## Defensive option (NOT recommended)

You could disable aud verification:

```python
options={"verify_exp": False, "verify_aud": False}
```

This works but defeats the new claim's purpose (cross-product token confusion mitigation). Prefer `audience=`.

## Version-skew note

ASM clients still running against the OLD server (pre-v0.22.0) won't have a `kid` claim and won't have an `aud` claim. The new code with `audience="asm"` still works against old tokens — `audience=` matches a present-`aud` claim, but pyjwt does NOT require the claim to be present when `audience=` is specified IF the token has no `aud`. **Wait — verify this.** pyjwt actually DOES require `aud` to be present when the caller passes `audience=`. Need to check. If pyjwt rejects tokens missing `aud` when `audience=` is set, the safer pattern is:

```python
import jwt

def _decode_license_jwt(token: str) -> dict:
    try:
        # Try with aud (v1.0+ server)
        return jwt.decode(
            token, _public_key_pem(),
            algorithms=["EdDSA"], audience="asm",
            options={"verify_exp": False},
        )
    except jwt.MissingRequiredClaimError:
        # Pre-v1.0 server fallback — no aud claim on the token
        return jwt.decode(
            token, _public_key_pem(),
            algorithms=["EdDSA"],
            options={"verify_exp": False, "verify_aud": False},
        )
```

This try/except pattern is safer for a rolling-upgrade scenario where ASM gets deployed before all tokens have rotated to the new server. If you're confident every cached token was minted post-v1.0 (e.g. after a forced phone-home on ASM startup), skip the fallback.
